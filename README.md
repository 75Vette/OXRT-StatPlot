# OXRT-StatPlot

Performance visualisation and analysis tool for OpenXR Toolkit CSV logs.

Created by **75Vette / The Beast**

---

# Overview

OXRT-StatPlot is a portable, zero-install Windows utility for analysing performance logs generated by the OpenXR Toolkit.

The tool reads `.csv` performance logs and generates high-quality visualisations of:

* FPS over time
* CPU App frametimes
* CPU Render frametimes
* GPU frametimes
* Frametime distribution histograms
* Overlay and split-view comparisons
* Automatic refresh-rate scaling
* Optional smoothing

Version 1.x introduces a full graphical user interface built with PySide6.

The application is fully portable and requires no installation.

## Plot Modes

### FPS Split Mode

One subplot per CSV log showing:
* FPS
* CPU App frametime
* CPU Render frametime
* GPU frametime

Automatic refresh-rate scaling and guide lines.

---

### FPS Overlay Mode

Overlay multiple FPS traces on a single graph.

Useful for:
* benchmark comparisons
* graphics settings comparisons
* driver comparisons
* hardware comparisons

---

### Frametime Distribution (Split)

Displays frametime distribution histograms for:
* GPU
* CPU App
* CPU Render

Using 0.1 ms frametime bins from 0–20 ms.

---

### Frametime Distribution (Overlay)

Overlay comparison mode for frametime distributions.

Useful for identifying:
* stutter behaviour
* frame pacing differences
* CPU vs GPU bottlenecks

---

## Automatic Refresh-Rate Detection

Automatically detects dominant headset refresh rate from FPS behaviour.
* 72 Hz
* 80 Hz
* 90 Hz
* 120 Hz
* 144 Hz

# Portable Usage

1. Download the latest ZIP release from GitHub.
2. Extract anywhere.
3. Run:
	OXRT-StatPlot-1.0.2.exe

---

# First Run Behaviour

On first launch, the application automatically creates:
	OXRT-StatPlot_Data/
next to the executable.

This folder stores:
	config.ini
	ui_state.ini
	Results/

---

# OpenXR Toolkit Stats Directory

By default, OXRT-StatPlot attempts to automatically locate OpenXR Toolkil stat .csv files here:
	%USERPROFILE%\AppData\Local\OpenXR-Toolkit\stats

If found, CSV logs are loaded automatically.

Custom locations can be configured in:
	Settings → Preferences

---

# Output Directory

Generated PNG files are saved into:
	OXRT-StatPlot_Data/Results/

Example filename:
	OXRT StatPlot 03-May-26 - 214522 - My Benchmark.png

---

# Configuration

The application automatically creates:
	config.ini
on first run.

---

# Key Configuration Options

## stats_dir = auto

Automatically uses the default OpenXR Toolkit stats folder.

---

## output_dir = auto

Uses:

```text
OXRT-StatPlot_Data/Results/
```

---

## auto_open_png = yes/no

Automatically opens generated PNG files after saving.

---

## verbose = yes/no

Enables debug logging to:

```text
engine.log
```
